Section I: Bezier Curves and Surfaces

Part 1: Bezier Curves with 1D de Casteljau Subdivision

De Casteljau's algorithm is a recursive algorithm used to evaluate and subdivide Bezier curves. Given n control points and a scalar parameter t (ranging from 0 to 1), we interpolate between each pair of adjacent control points to produce n - 1 new points. Repeating this process until only one point remains gives us the point on the Bezier curve at parameter t.

To implement the algorithm, I completed evaluateStep and evaluate1D. evaluateStep performs a single level of the recursion — it iterates through each adjacent pair of points in the input and computes the interpolated point (1 - t) * p[i] + t * p[i+1], collecting results into a vector res. evaluate1D does the recursive work by repeatedly calling evaluateStep until only one point remains.

Part 2: Bezier Surfaces with Separable 1D de Casteljau

Bezier curves are 1D lines in 2D space. Bezier surfaces are 2D patches in 3D space. Rather than being defined by a set of n points, surfaces are defined by an n × m grid of control points. We evaluate a Bezier surface using de Casteljau's algorithm by treating the surface as a series of Bezier curves. For each row of control points, we apply the 1D algorithm using parameter u to reduce it to a single point. These resulting points form a new curve, which we then evaluate using parameter v.

To implement this, I completed evaluate. For each row, evaluate1D is called to produce a single 3D point using parameter u. The results are collected into rowRes, so that it contains one point per row. This set of points is then reduced to the final surface point by calling evaluate1D(rowRes, v).

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

I started by walking the vertex's one-ring using the halfedge data structure (vIter->twin()->next()). For each adjacent triangle, I took its three vertex positions (pos0, pos1, pos2) and computed the cross product of two edge vectors: cross(pos1 - pos0, pos2 - pos0). This gives a vector perpendicular to the triangle whose length is proportional to the triangle's area. I accumulated all of these into n, then normalized at the end to get a unit normal. The area-weighting happens automatically because larger triangles produce longer cross-product vectors, contributing more to the final sum.

Part 6: Loop Subdivision for Mesh Upsampling

The implementation is split into five steps:

  1. Compute new positions for original vertices. For each vertex, I set isNew = false and apply the Loop weighting rule: (1 - n*u) * pos + u * neighborSum, where u = 3/16 for degree-3 vertices and 3/(8n) for all others. Results are stored in newPosition.
  2. Compute new positions for edge midpoints. For each edge, I set isNew = false and compute 3/8 * (v0 + v1) + 1/8 * (v2 + v3), where v0 and v1 are the edge's endpoints and v2, v3 are the opposite triangle vertices. Results are stored in Edge::newPosition.
  3. Split all original edges. I snapshot nEdges() before splitting to avoid an infinite loop — without this, the loop would process newly created edges and split them too. For each original edge, I call mesh.splitEdge(e) and copy the precomputed e->newPosition onto the new midpoint vertex.
  4. Flip crossing new edges. Any new edge that connects exactly one old vertex and one new vertex is flipped. This restores the correct Loop subdivision topology.
  5. Apply new positions. Copy newPosition into position for all vertices.

Instead of computing positions in-place, I computed and stored all new positions before any splits or flips. Doing the computations in-place corrupted neighbor lookups mid-loop.

Observations on mesh behavior after Loop subdivision:

  1. The mesh becomes smoother as subdivision level increases.
  2. The size of the object shrinks very slightly with each subdivision level.
  3. The number of triangles multiplies by 4 each subdivision level.
  4. Sharp corners and edges become more rounded as subdivision level increases.

Cube asymmetry and pre-processing. The cube subdivides asymmetrically without pre-processing because each face has only one diagonal edge, giving different corners different vertex degrees. Corners that have more diagonals pointing toward them have higher degree than corners with fewer. When we apply Loop subdivision, vertices are weighted based on their degree in the formula (1 - n*u) * pos + u * neighborSum — vertices with different degrees are pulled inward by varying amounts, producing the asymmetry we observe.

To fix this, I split each diagonal edge so that every face has an X pattern (both diagonals). Under this new topology, each face is identical (two diagonals) and each corner is identical (degree 6). When we perform Loop subdivision, each vertex is weighted equally and pulled inward by the same amount, resulting in a symmetric rounded shape.
